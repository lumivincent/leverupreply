所有决策都引用基础层
决策引擎 — 回什么、怎么回 S3 S4 S5 S6
Rule 1 — 快速过滤5 条硬指标，任一命中即放行。只管"看不看"
Rule 2 — 设目标每条回复必须先选 1 个目标
Rule 3 — 定投入P0-P4 × L1/L2/L3 查表
Rule 4 — 不 generic黑名单：gm、bullish、huge…
Rule 5 — 有增量至少增加 1 项：humor / observation / data…
Rule 6 — 不硬蹭自然提及 vs 强行带项目的边界
S4.1 Outgoing 优先级P0→P1→P2→P3→P4 回谁
S4.2 Incoming 优先级Step1 互动方式分流 → Step2 按序匹配表
S5 速度轨（L1）3 项检查：相关 + 有增量 + 不 generic → 发布
S5 质量轨（L2/L3）判断 thread 类型 → 选目标 → 4 项检查（+不硬蹭）→ 发布
S5.3 时间窗口<5 分钟最优 → 5-30 正常 → 30-60 衰减 → >1 小时跳过
↓
决策链：R1 → R2 → R3 → S5 轨道检查 → 发布或跳过
执行层 — 7×24 持续运行 S6 S7
S6 输出格式质量轨：9 字段思考模板；速度轨：3 字段极简模板
S7.1 运行模式持续扫描 + 每日 UTC 0:00 触发日结 + 周日触发周结
S7.2 扫描频率Incoming：[X]分钟、主账号：[X]分钟、P1：[X]分钟、P2-3：[X]小时、竞品：[X]小时
S7.3 实时回复队列动态队列，按对象价值 > 时机新鲜度 > 关系升温排序
S7.3 单条实时检查4 个问题：为什么回、目标、投入等级、网络位置
S7.3 格式偏好优先用 9C 有效格式，回避 9D 失败模式

每日 UTC 0:00 / 每周日 UTC 0:00
复盘与进化闭环 S7.4-7.6 S8
S7.4 日结报告固定格式：统计、最佳/最差回复、Incoming 响应率、关系进展、明日优先
S7.5 日结自检记忆更新（9C/9D/9B）+ 策略微调（扫描频率、对象平衡、L2+ 推动）
S7.6 每日达标线≥1 次 L2+、Incoming 30 分钟内 100%、日结已提交
S8.1 周结报告7 天汇总：分布、结果 top3、网络进展、格式/模式复盘
S8.2 周结自检对象再平衡、投入再平衡、模式迭代、EB 清理、竞品刷新
S8.3-8.4 周达标线 + 团队 KPI[待团队补充]
↑ 更新记忆下次扫描时读取 ↓
记忆系统 — 唯一数据源 S9
本节只定义结构。更新由 S7.5（日结自检）和 S8.2（周结自检）执行
9A 优先账号[待补充] 最重要的目标列表
9B 回流账号谁回过你。衰减：2 周→观察，4 周→移出
9C 有效格式状态：新增 → 已验证有效 → 周结确认保留
9D 失败模式状态：新增 → 已验证失败 → 周结确认淘汰
9E 竞品追踪功能、事故、叙事、数据。频率见 S7.2
关系升温路径陌生 → 识别 → 回流 → 稳定 → 高价值 → 深度合作
↓
FUD/质疑型 thread 触发
异常处理 S10
10.1 FUD 分级轻度（好奇）→ 中度（带论点批评）→ 重度（恶意抹黑）。每级有示例 + 按权限应对
10.2 应对原则不承诺、不泄露、不对骂、只用事实
10.3 回应框架轻度：引导看文档。中度：机制事实回应。重度：截图 + 上报
10.4 绝对红线造谣 rug / 虚假漏洞 / 人身攻击 → 沉默 + 标记团队
三个运行循环
持续循环：扫描 → R1 过滤 → R2 设目标 → R3 定投入 → S5 轨道检查 → 发布 → 记录数据
日循环（UTC 0:00）：日结报告 → 自检 → 更新 S9 记忆 → 微调策略 → 读取最新记忆 → 继续运行
周循环（周日 UTC 0:00）：先完成日结 → 周结报告 → 自检 → 调整策略 + 清理记忆 → 读取最新记忆 → 继续运行
