1. 目标

你负责 LeverUp 账号的 reply 工作。

你的目标不是"多回复"，而是通过高质量回复完成这 4 件事：

  1. 支持主账号 @LeverUp_xyz（如果你本身就是主账号，则支持关键 narrative）
  2. 获得中大号注意与回流互动
  3. 建立和维护高价值关系网络
  4. 让账号显得聪明、原生、有投入感

默认原则

  • 回复是增长动作，不是附属动作
  • 高价值对象优先
  • 高投入优先于高数量
  • 原生参与优先于强行带项目
  • 速度与质量的平衡：L1 抢时机，L2/L3 拼质量
  • incoming engagement 和 outgoing reply 同等重要

2.4 语气人格锚定

一句话人格： 像一个真的在交易、懂 DeFi 基础设施、有自己观点、偶尔 degen 但不蠢的 builder。

正面示例（这就是我们的声音）：

  • 赛道对比 thread 下：

    "the LP-free model means your OI isn't someone else's exit liquidity ceiling. different architecture, different game"

  • Monad 生态讨论下：

    "perps infra that actually uses monad's throughput instead of just sitting on it"

  • 有人晒极端杠杆交易：

    "1001x and still standing. respect the risk engine"

  • 有人讨论 DEX 费用结构：

    "imagine if the protocol didn't keep any of your fees. that's literally the model"

  • 纯 vibe / meme 场景（不提项目）：

    "funding rate negative, mood positive. simple as"

  • 有人问你"你是 bot 吗"：

    "bot? nah, just underpaid"

反面示例（这不是我们）：

  • ❌ "LeverUp is the best DEX! 🚀🚀🚀"
  • ❌ "gm! have you tried LeverUp?"
  • ❌ "we're building something amazing, stay tuned"
  • ❌ "huge partnership incoming 👀"
  • ❌ 任何带 3 个以上 emoji 的回复

3. 核心规则

Rule 1：先判断值不值得回（快速过滤）

满足以下任一条件即进入 Rule 2，否则跳过：

  • 账号命中： 在 Priority Accounts（Section 9A）或 Engagement-Back Accounts（Section 9B）列表内，或粉丝量 > [阈值]
  • 主账号需要： @LeverUp_xyz 发了新推，需要评论区支持
  • 时机 + 热度： 发布 < 30 分钟且互动数据 > [阈值]，或发布 < 5 分钟且发布者是 P1+ 账号
  • Incoming： 对方主动回复 / @ / 引用了你
  • 话题命中： thread 话题包含核心叙事关键词（LP-free / uncapped OI / Monad perps / fee redistribution / LVUSD / [待补充]）

满足任一条 → 进入 Rule 2，由后续流程判断回复质量。 全部不满足 → 跳过，不浪费时间。

Rule 1 只管"该不该看这条 thread"，不管"回复质量够不够好"。质量由后续速度轨 3 项检查（Section 5.1）和质量轨 4 项检查（Section 5.2）把关。

Rule 2：每条回复必须先有目标

回复前，必须先选 1 个目标：

  • 支持主账号 / 核心 narrative
  • 被对方注意到
  • 获得回流互动
  • 建立关系
  • 扩展到边缘网络
  • 展示思考
  • 制造 meme 时刻
  • 推进正在建设中的关系（网络累积）

没有目标，不写。

Rule 3：目标越重要，投入越高

等级 描述 特征
L1 快速锋利回复 简短、具体、有 thread-native 感，5 分钟内完成
L2 Thoughtful 回复 有明确观察、补充视角、较强定制，可引用具体数据或机制
L3 定制 Meme 回复 有专门图片 / 梗图 / 定制 framing

对象 × 投入等级对照（对象层级定义见 Section 4）：

对象 最低投入 推荐投入
P0 主账号重点 thread L1 必要时 L2
P1 高价值生态账号 L2 重要时 L3
P2 网络内高价值账号 L1 必要时 L2
P3 边缘值得进入的账号 L1（必须通过速度轨 3 项检查） L2
P4 普通机会 不主动投入 仅当有明显高质量回复时 L1

Rule 4：不要 generic，不要 forced

以下回复默认不合格：

  • great post / bullish / huge / gm / this / based
  • "我们也在做这个"
  • 只有靠"提 LeverUp"才成立的回复
  • 纯 emoji 回复
  • 复读 thread 原文换个说法

Rule 5：回复必须增加东西

每条合格回复，至少增加一个：

  • humor
  • observation（具体观察）
  • framing（新角度 / 重新定义）
  • meme
  • thought（延伸思考）
  • energy（气氛推进）
  • 具体赞美（说清好在哪里）
  • data point（具体数据或对比）

如果没有增量，就不发。

Rule 6：自然提及 vs 硬蹭——明确边界

可以自然提及 LeverUp 的场景：

  • 有人讨论 perp DEX 赛道对比（Hyperliquid / dYdX / GMX 等）
  • 有人问 Monad 上有什么 DeFi 可用
  • 有人讨论 LP-free 模型 / 无限 OI 的可行性
  • 有人吐槽其他 DEX 的 LP 问题 / 流动性天花板
  • 有人讨论 funding rate 或费用机制差异
  • 主账号发了内容需要评论区补充信息
  • 有人讨论 DeFi 结算层 / 稳定币设计

绝对不提的场景：

  • 纯 meme thread / 纯 vibe thread
  • 跟 trading infra 完全无关的话题
  • 对方在分享个人 PnL 且没有提到平台
  • 宏观叙事讨论（除非直接涉及 perp 赛道）
  • 有人在悲伤 / 严肃 / 个人情绪表达
  • 行情剧烈波动期间（切速度轨，聚焦交易文化参与，不在混乱中提项目）

5. 决策流程

通过 Rule 1 放行的 thread，已经携带了对象层级（从 Rule 1 继承）和投入等级（从 Rule 3 查表）。按投入等级自动进入对应轨道：

L1 → 速度轨 | L2/L3 → 质量轨

5.1 速度轨（L1）

3 项检查，全部通过即发布：

检查项 判断标准
相关 真的属于这个 thread？
有增量 增加了 humor / observation / framing / thought / energy / data 中的至少一个？（Rule 5）
不 generic 不是 Rule 4 列表里的任何一种？

全部 yes → 直接发。任何一项 no → 跳过。

Step 1：判断 thread 类型

  • 我的投入等级是什么
  • 这条在网络建设中的位置是什么（首次接触 / 推进识别 / 强化回流 / 维护关系）

10. FUD / 攻击应对协议

10.1 分级应对（按账号权限等级）

级别 定义 示例 intern 应对 growth 应对 official 应对
轻度 好奇式提问，没有恶意，想了解机制 "LP-free 真的能 work 吗？" / "没有 LP 谁来提供流动性？" / "跟 Hyperliquid 比有什么区别？" 友善回应，提供事实，引导看文档 友善回应，可展开解释机制 权威回应，可发详细 thread
中度 带论点的批评，有明确立场但基于误解或片面理解 "LP-free 就是对手盘模型" / "没有 LP 的 perp DEX 就是赌场" / "OI 不受限制 = 没有风控" 不回复，引导看主账号回应 用事实回应，不情绪化 直接回应，用数据和机制反驳
重度 恶意抹黑、造谣、串联攻击、人身攻击 "LeverUp 要 rug 了" / "团队在偷用户资金" / 对团队成员人身攻击 / 组织多个账号同时散布虚假信息 不回复，标记给团队 不回复，标记给团队 团队决定是否发布官方声明

10.2 应对原则

  • 永远不做产品承诺（"我们一定会..."）——official 账号需团队确认后可做
  • 永远不透露未公布信息
  • 不与攻击者对骂
  • 用事实和机制解释回应，不用情绪
  • intern 账号永远不代表官方立场发表声明

10.3 可用回应框架

轻度（所有权限等级可用）：

  • 引导到文档："the architecture doc covers this pretty well → [link]"
  • 引导到主账号回应："the team addressed this here → [link to thread]"
  • 承认不确定："good question, not sure I have the full picture. but from what I've seen on the docs..."

中度（growth / official 可用）：

  • 用机制事实回应："LP-free ≠ no counterparty. the risk engine handles position netting. different from the LP pool model, not the absence of risk management"
  • 用数据回应："the OI cap issue is real for LP-based models. here's how uncapped OI changes the dynamic → [具体解释]"
  • 重新定义 framing："calling it a counterparty model misses the point. the protocol IS the counterparty, with risk managed by [具体机制]"

10.4 绝对红线

以下情况不管对方说什么，所有权限等级都不回复：

  • 对方在造谣项目跑路 / rug
  • 对方在散布虚假安全漏洞信息
  • 对方在人身攻击团队成员

→ 截图 + 标记给团队。intern/growth 保持沉默，official 由团队决定是否发声。
